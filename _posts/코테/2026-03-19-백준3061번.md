## 풀이

### 1. 사다리 가로줄의 동작 분석

사다리의 가로줄은 인접한 두 경로를 연결하여 순서를 교환하는 역할을 한다.

```text
[1] [2] [3] [4]
     ├──┤          ← 가로줄
[1] [3] [2] [4]       2번과 3번 위치 교환
```

* 가로줄이 하나 추가될 때마다 인접한 두 원소 간의 위치 교환(swap)이 정확히 1회 일어난다.

### 2. 입력 데이터 해석

입력 예제 `3 2 4 1`은 각 위치에 도달하는 시작점 번호를 의미한다.

* 이를 각 번호별 최종 위치로 변환하면 다음과 같다.

| 원소 | 최종 위치 |
|------|-----------|
| 1번  | 3번째     |
| 2번  | 2번째     |
| 3번  | 4번째     |
| 4번  | 1번째     |

* 이를 바탕으로 최종 상태를 나열하면 `4 2 1 3`이 된다.

### 3. 교환 횟수와 역전 쌍의 관계

원소가 이동하는 과정을 분석했다. 4번 원소가 4번째에서 1번째 위치로 이동하는 과정:

```text
[1] [2] [3] [4]   ← 초기 상태
[1] [2] [4] [3]   ← 4와 3 교환
[1] [4] [2] [3]   ← 4와 2 교환
[4] [1] [2] [3]   ← 4와 1 교환
```

* 4번이 3칸을 이동하기 위해 3번의 인접 교환이 발생했다.
* 이 과정에서 `(1,4), (2,4), (3,4)` 세 쌍의 상대적인 순서가 역전되었다.

여기서 초기 상태에서는 `a`가 `b`보다 앞에 있었으나, 최종 상태에서는 `b`가 `a`보다 앞에 위치하게 된 상태를 **역전 쌍(Inversion)**으로 정의했다.

가로줄 1개는 인접한 두 원소의 순서를 1번 뒤집으므로, 다음과 같은 규칙을 도출했다.

* 역전 쌍 1개를 원래 상태로 되돌리기 위해 최소 1번의 교환이 필요하다.
* 1번의 인접 교환은 정확히 1개의 역전 쌍 상태를 변경한다.
* 따라서 확인된 **전체 역전 쌍의 개수**가 문제에서 요구하는 **최소 가로줄 수**와 일치한다.

## 코드

```kotlin
import java.io.BufferedReader
import java.io.InputStreamReader

fun main(): Unit = with(BufferedReader(InputStreamReader(System.`in`))) {
    val t = this.readLine().toInt()
    val sb = StringBuilder()

    repeat(t) {
        val n = this.readLine().toInt()
        val dl = this.readLine().split(" ").map { it.toInt() - 1 }.withIndex().sortedBy { it.value }.map { (i, v) -> i }


        var answer = 0
        for (i in 0 until n) {
            for (j in i + 1 until n) {
                if (dl[i] > dl[j]) answer++

            }
        }
        sb.append("$answer\n")
    }
    println(sb.toString())
}
```
