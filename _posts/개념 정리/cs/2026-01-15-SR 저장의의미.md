디지털 회로를 공부하면서 SR 래치가 "상태를 저장한다"는 표현을 계속 마주쳤다. 처음에는 이해가 잘 되지 않았다. 논리 게이트는 입력에 따라 출력이 결정되는 단순한 구조라고 생각했는데, "저장"이라는 개념이 어떻게 들어가는지 명확하지 않았다.

## 일반 논리 회로의 동작

AND, OR, NOT 같은 기본 논리 게이트는 입력이 같으면 출력도 항상 같았다. AND 게이트에 (1, 1)을 넣으면 언제나 1이 나왔고, 과거에 어떤 입력을 넣었는지는 결과에 영향을 주지 않았다. 현재 입력 값만으로 출력이 완전히 결정되는 구조였다.

## SR 래치의 차이

SR 래치는 달랐다. 같은 입력을 넣어도 출력이 달라질 수 있었다.

예를 들어 S=1, R=1을 입력했을 때:

- 직전 상태가 S=0, R=1이었다면 출력은 1
- 직전 상태가 S=1, R=0이었다면 출력은 0

같은 입력인데 과거 상태에 따라 출력이 달라졌다. 이게 일반 논리 게이트와의 핵심적인 차이였다.

## 저장의 의미

S=1, R=1 상태는 "이전 값을 유지하라"는 신호였다.

- S=0, R=1 → 출력을 1로 설정
- S=1, R=1 → 출력 1 유지
- S=1, R=0 → 출력을 0으로 설정
- S=1, R=1 → 출력 0 유지

S=0 또는 R=0으로 값을 쓰고, 다시 S=1, R=1로 돌아오면 그 값이 유지되었다. 이것이 "저장"이라고 부르는 동작이었다.

## SR 래치의 세 가지 상태

S=1, R=1 상태가 "저장"을 의미한다는 것을 알게 되면서 SR 래치의 동작이 명확해졌다.

- **S=0, R=1**: 1을 쓴다
- **S=1, R=0**: 0을 쓴다
- **S=1, R=1**: 이전 값 유지 (저장 상태)

S=1, R=1은 "아무것도 건드리지 마"라는 신호였다. 이 상태에서 출력 a는 마지막에 쓴 값을 계속 유지했다.

## 저장된 값을 사용하는 방법

"꺼낸다"는 특별한 동작이 필요하지 않았다. 출력 a는 항상 저장된 값을 내보내고 있었고, 이 출력을 다른 회로의 입력으로 연결하면 그게 읽기였다.

``` text
[SR 래치] ──출력 a──┐
                    ├──→ [AND 게이트] ──→ 결과
[다른 입력 B] ──────┘
```

## 저장 기능의 필요성

컴퓨터는 중간 결과를 저장해야 복잡한 계산을 할 수 있었다. 예를 들어 1+2+3을 계산하려면, 1+2의 결과인 3을 어딘가에 저장해뒀다가 다시 꺼내서 +3을 해야 했다. 저장 기능이 없으면 이전 결과를 기억하지 못해서 한 번에 하나의 계산만 하고 끝났다.

## 휘발성 메모리인 이유

SR 래치는 S=1, R=1 상태에서 전기가 계속 공급되어야 값을 유지했다. 컴퓨터 전원이 꺼지면 피드백 루프가 멈추고 저장된 값이 사라졌다. 이것이 RAM이 휘발성 메모리인 이유였다.
