## Design Context

### Users

한국어 개발자 본인과 비슷한 관심사를 가진 기술 독자.

주로 **긴 기술 글**(네트워크·쿠버네티스·Spring·Elasticsearch·Redis Streams 등 이미 68개 이상)을 TOC·코드블록·다이어그램·LaTeX 수식을 오가며 읽는다. 데스크톱 비중이 높고, 다크 모드에서 장시간 읽는 세션이 많다.

동기는 이중 구조다:
1. **Author** (본인): 학습·트러블슈팅 기록 아카이브, 그리고 React 19 실험장
2. **Reader** (비슷한 개발자): 문제 해결 중 우연히 닿아서 깊게 읽음

이 사이트는 "읽기 도구"이면서 동시에 "작가의 작업장"이라는 이중 성격을 갖는다. 둘 다 만족해야 한다.

### Brand Personality

**Sharp · Honest · Playful**

- **Sharp** — **타이포그래피의 날카로움**. 정확한 위계, 군더더기 없는 레이아웃, 선명한 텍스트 대비
- **Honest** — 엔지니어 직설체. "이 부분이 함정이야" 같은 톤. 과장·불필요한 형용사 없음
- **Playful** — 지면 위 장식의 장난이 아닌, **언어와 디테일의 장난**. 문장·링크·마이크로 인터랙션 수준에서만 드러남

**감정 목표**: *"이 사람 진짜 잘 알고 있네"* + *"텍스트만 있는데 왠지 기분 좋다"*. 권위 있는데 딱딱하지 않음.

**말투 예시**
- ❌ "본 문서에서는 Kubernetes의 Blue-Green 배포 전략에 대해 심층적으로 다루고자 합니다."
- ✅ "Blue-Green 배포는 생각보다 간단한데, k8s에서 하려면 한 가지 함정이 있다. 이걸 먼저 보자."

### Aesthetic Direction

**Refined Editorial Dark · Serif Display + Inter body · Electric Blue accent**

세 개의 레퍼런스 — **현재의 Gumroad · Linear · impeccable.style** — 가 공유하는 **editorial 캐릭터**를 따른다. 순수 미니멀(장식 제로)이 아니라, **"장식은 줄이되 캐릭터는 유지"**하는 미니멀.

#### 핵심 원리

- **위계는 타이포가 만든다** — 하지만 Inter 단일 폰트만으로는 약하다. **Serif display + sans body 이중 스택**으로 editorial 에너지를 얻음
- **카드는 얇게 존재** — 완전 투명이 아니라 `1px solid var(--border)` + 은은한 `--bg-subtle` 배경 + `border-radius: 10px`. Gumroad·impeccable.style 둘 다 이 패턴
- **Cards-on-cards는 여전히 금기** — 카드 안에 또 박스 금지
- **섹션에 번호** — CSS 카운터로 `01`, `02`, `03` editorial 번호를 카테고리 섹션 앞에 자동 생성 (impeccable.style의 "01 The Antidote" 패턴)
- **거대한 제목** — 포스트 상세 h1은 3rem (48px), 카테고리 페이지 타이틀 2.75rem. 제목이 지면을 지배해야 editorial

#### 색 전략

- 배경 3층 레이어: `--bg: #0a0a0b` / `--bg-subtle: #111116` (카드) / `--bg-hover: #14141a` (호버). 차이는 미세하지만 카드를 배경에서 떠오르게 함
- 텍스트 다층 투명도 (`--text` / `--text-secondary` / `--text-muted` / `--text-faint`)
- 테두리 `#1f1f24` 조용하게 (`--border-hover: #2b2b33`으로 호버 시 살짝 올림)
- **Electric Blue `#2b5bff`는 딱 네 곳에만**: 링크, 포커스 아웃라인, 인라인 코드, hover 시 제목 색
- **그림자는 여전히 쓰지 않음**

#### 타이포 전략 (이중 스택)

- **Display (제목·큰 타이포)**: **Fraunces** (variable serif, opsz/wght 축) → Korean은 **Noto Serif KR**로 자동 fallback
  - Fraunces는 impeccable.style의 "The Antidote" 같은 editorial serif 느낌
  - `font-optical-sizing: auto`로 크기별 최적 렌더링
- **Body·UI**: **Inter** → Pretendard Variable (한글) fallback
- **Code**: JetBrains Mono
- 본문 17px · line-height 1.8 · `word-break: keep-all` (**한국어 가독성 절대 양보 없음**)
- 제목이 serif, 본문이 sans 로 **극적인 대비**. 이게 editorial 사이트의 핵심 문법

#### 인터랙션

- Hover: **색 변화 + 카드 배경 미세 lift** (`--bg-subtle` → `--bg-hover`, 테두리도 한 단계 위로). translate·scale은 없음
- 예외: 테마 토글 active 시 `scale(0.96)` (단 하나의 촉각 피드백)
- `prefers-reduced-motion` 준수

#### 참조 결

- **impeccable.style** (https://impeccable.style) — 이 방향의 **일순위 레퍼런스**. 서체 선택(Fraunces/Instrument Serif 계열), 섹션 번호 `01`, DO/DON'T 레이아웃, 핑크 액센트 → 우리는 블루 액센트로 대체
- **Linear** (https://linear.app) — 다크 베이스의 기준점. 거대한 editorial 헤드라인, 레이아웃 여백 감각
- **Gumroad** (https://gumroad.com, 현재) — 얇은 카드 보더 + 둥근 radius + 카드 안 콘텐츠의 캐릭터

이 세 사이트가 공유하는 건 "캐릭터 없는 미니멀"이 아니라 **"캐릭터는 유지하면서 장식만 줄인 editorial 미니멀"**이다. 2026-04-05 두 번째 시도(Refined Minimal, Inter 단일, 박스 제거)는 이 구분을 놓쳐서 너무 스트립됐다는 사용자 피드백을 받았다. 이 파일의 현재 버전은 그 교정 후 3차 시도.

### 안티 레퍼런스 (명시적 금기)

`docs/specs/layout.md §2.3`에 원래 있던 것들에 더해, 2026-04-05 같은 날 두 번 시도하고 기각한 것들:

**원래 있던 것**
- 회색 + 파란색 accent + 둥근 카드 + 가벼운 soft shadow (제네릭 AI 룩)
- 무의미한 그라데이션, Glassmorphism, 블러
- 장식용 일러스트·이모지 과사용
- Material Design·Bootstrap 기본 룩

**2026-04-05 1차 시도 기각: Polished Brutalism**
- **2px 하드 테두리 + 오프셋 컬러 섀도우(`6px 6px 0 #2b5bff`)**
  - 기각 이유: 레퍼런스 셋(현 Gumroad / Linear / impeccable.style)이 전부 이 방향과 반대였음. Gumroad 2019~2022 브루탈 시절 기억이 낡아서 잘못 해석
- **Cards-on-cards** 패턴 (`.nav-index` 박스가 `.nav-index__chip` 박스를 품는 구조)
  - 기각 이유: impeccable.style이 명시적으로 경고한 패턴

**2026-04-05 2차 시도 기각: Refined Minimal (너무 스트립)**
- **완전히 투명한 Chrome** — 카드 보더 전부 제거, 배경 틴트 없음, radius 0, 수평선만으로 구분
- **Inter 단일 폰트 패밀리** — 제목·본문·UI 동일 폰트, 위계는 크기만으로
- **섹션 번호·키커 라벨 없음** — 장식을 전부 제거하다 보니 editorial 캐릭터까지 제거됨
  - 기각 이유: 사용자가 레퍼런스 이미지(Gumroad 카드 + Linear 큰 제목 + impeccable.style serif + 섹션 번호)를 보여주며 "너무 다르다"고 피드백. "미니멀"을 **"장식 제로"**로 해석한 게 원인. 실제 레퍼런스들은 **"장식은 줄이되 캐릭터는 유지"**. 얇은 카드 보더·editorial serif·섹션 번호는 장식이 아니라 editorial 문법
- 이 시도의 교훈: **"minimal"과 "character-less"는 다르다.**

**현재 (3차): Refined Editorial Dark** — 1차의 캐릭터 욕망 + 2차의 restraint를 모두 반영. 서체는 impeccable.style에서, 카드는 Gumroad에서, 레이아웃·여백은 Linear에서.

**낡은 결정을 지우지 않는 이유**: CLAUDE.md 규칙 #6 — 번복 이유가 다음 번복을 막는다. 이 세 시도의 타임라인이 보존되어야 미래에 "어, 브루탈 다시?" "어, 더 스트립?" "어, 서체 더 장식적?" 등이 나왔을 때 이미 왜 어떤 방향이 기각됐는지 즉시 볼 수 있다.

### Design Principles

#### 1. Editorial이란 "캐릭터 + 여백"이지 "장식 + 여백"이 아니다
포스트 제목을 **거대한 serif(Fraunces)**로 쓰는 건 장식이 아니라 **editorial 문법**이다. 섹션 앞에 `01`, `02` 번호를 붙이는 것도, 얇은 카드 보더도 같다. 이런 것들은 장식이 아니라 "이건 editorial 사이트야"라는 언어다. 제거하면 editorial이 아니라 단순 미니멀이 된다. **미니멀과 캐릭터 없음은 다르다.**

#### 2. 타이포그래피는 이중 스택
제목은 **Fraunces + Noto Serif KR** (editorial serif), 본문·UI는 **Inter + Pretendard** (sans). 이 대비가 editorial 사이트의 핵심 문법. 단일 폰트로 위계를 만들려 하면 약해진다. (2차 시도에서 이걸 배웠다.)

#### 3. `#2b5bff`는 딱 네 곳에만
링크, 포커스 아웃라인, 인라인 코드, hover 시 제목 색. 그 외에는 쓰지 않는다. 그림자·테두리·대형 배경에 블루가 들어가는 순간 **벽지**가 된다.

#### 4. 카드는 **조용한 박스**
카드는 박스를 가지되 조용해야 한다: `1px solid var(--border)` + `--bg-subtle` 배경 + `radius 10px` + 섀도우 없음. Hover 시 배경·보더 한 단계씩만 lift. Cards-on-cards는 여전히 금기.

#### 5. 움직임은 색·미세한 배경 lift만
Hover 피드백은 **색 변화 + 카드 배경 lift** (subtle→hover). translate/scale/shadow 변화는 쓰지 않는다 (단 하나의 예외: 토글 클릭 시 `scale(0.96)`). `prefers-reduced-motion` 준수.

#### 6. 한국어 가독성은 절대 양보 없음
본문 17px · line-height 1.8 · `word-break: keep-all` · WCAG AA 대비 유지. 한글 제목은 Noto Serif KR(editorial serif 매칭) 또는 Pretendard(fallback)로 렌더. 어떤 방향이든 긴 기술 글 읽기 세션이 편해야 한다.
