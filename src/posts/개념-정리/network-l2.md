## NIC

**NIC(Network Interface Card)** 는 물리적으로 네트워크에 연결되는 **인터페이스 장치**다. 흔히 '랜 카드'라고 부르지만, 전문 용어로는 NIC 또는 네트워크 인터페이스라고 한다.

### 1. 주요 특징

- **하드웨어 장치**
    - 유선 NIC(이더넷 포트)와 무선 NIC(Wi-Fi 모듈)이 있으며, 구분 없이 모두 NIC로 지칭
- **MAC 주소**
    - NIC마다 고유의 MAC 주소를 가짐
    - 한 호스트에 여러 개 NIC가 있으면 MAC 주소도 그만큼 존재
- **호스트와의 관계**
    - 하나의 호스트(컴퓨터)는 여러 NIC를 장착 가능
    - 네트워크 도식에서는 NIC를 진한 점으로 표시하며, 점이 여러 개면 인터페이스가 여러 개 존재함을 나타냄

**예시:** 노트북 내장 Wi-Fi 모듈 + USB 유선 랜카드를 연결 → 총 2개의 NIC가 활성화

## MAC 주소

**MAC 주소**는 **NIC**를 고유하게 식별하는 **48비트 식별자**다.

- **표현 형식**
    - 16진수 6쌍, 예: AB-CD-EF-12-34-56
    - 각 쌍은 8비트(1바이트)에 해당
- **용도**
    - 이더넷, Wi-Fi 등 **데이터 링크 계층** 통신에서 호스트 식별
    - 스위칭 장비가 목적지를 결정할 때 사용

### 1. MAC 주소 중복 방지 메커니즘

1. **48비트 공간의 방대함**
    - 전체 주소 공간은 2^48개(약 281조 개)로, 단일 네트워크 내에서도 사실상 주소 고갈이 일어나기 어렵다.
2. **OUI(Organizationally Unique Identifier) 제도**
    - MAC 주소의 **앞 24비트**는 IEEE가 **제조사별로 고유하게 배정**하는 값
    - 예를 들어, A에는 00-1A-2B가, B에는 00-1A-2C가 할당되는 식으로 관리된다.
3. **제조사 내부 관리**
    - 각 제조사는 **뒤 24비트**의 주소를 자사 장비별로 고유하게 배정한다.
4. **IEEE 등록 시스템**
    - 모든 OUI 할당 내역은 **IEEE 공식 데이터베이스**에 등록·공개되어, 외부에서도 확인 가능하다.
    - 등록되지 않은 OUI를 사용하면 **규격 위반**으로 간주되어 시장 진입이 제한된다.

### 2. MAC 주소 변경 가능 여부

NIC 하드웨어에는 ROM으로 고정된 원래 MAC 주소가 있다. 이를 **BIA(Burned-In Address)** 라고 한다. 이 값 자체는 변경되지 않는다.

다만 OS 레벨에서 프레임에 넣는 출발지 MAC 주소를 **덮어씌우는 것**은 가능하다. 하드웨어의 원본 MAC은 그대로 남아 있고, OS가 패킷을 보낼 때 소프트웨어에서 설정한 MAC을 대신 사용하는 방식이다.

- **Linux**: `ip link set dev eth0 address XX:XX:XX:XX:XX:XX`
- **Windows**: 네트워크 어댑터 속성에서 "네트워크 주소" 값 변경
- **macOS**: `ifconfig en0 ether XX:XX:XX:XX:XX:XX`

이를 **MAC 스푸핑(MAC Spoofing)** 이라고 부르며, 보안 테스트나 네트워크 디버깅에서 사용되기도 한다. 악용 방지를 위해 일부 네트워크에서는 **포트 보안(Port Security)** 으로 특정 MAC 주소만 허용하도록 제한하기도 한다.

## L2 Frame (Ethernet Frame)

**L2 Frame**은 **데이터 링크 계층(Layer 2)**에서 사용되는 데이터 전송 단위다.
가장 일반적인 형태는 Ethernet Frame이며, 다음과 같은 구조를 갖는다.
- **출발지 주소**
- **목적지 주소**
- **Payload**

## LAN (Local Area Network)

네트워크는 범위에 따라 WAN, MAN, LAN 등의 통신망으로 구분된다. WAN은 광역 통신망, MAN은 도시권 통신망을 의미한다.
LAN 통신을 위해 사용되는 가장 대표적인 통신은 Ethernet과 Wi-Fi다.

### 1. 예시 환경

- 가정 내 네트워크: 공유기를 통해 PC, 스마트폰, 태블릿 등이 연결됨
- 회사나 학교: 수십~수백 대의 장비들이 유기적으로 연결됨

## L2 스위치: Access vs Distribution

**L2 스위치**는 데이터 링크 계층(Layer 2)에서 작동하며, MAC 주소를 기반으로 데이터를 전달하는 장치다.
네트워크 규모나 역할에 따라 **Access 스위치**와 **Distribution 스위치**로 구분된다.

### 1. L2 Access 스위치

Access 스위치는 **엔드 포인트 장비**와 직접 연결되는 말단 스위치다.

#### 특징

- **MAC 주소 기반 스위칭**
    - 각 포트에 연결된 장비의 MAC 주소를 테이블에 저장하고, 이를 기반으로 데이터를 전달
- **포트 (Port) / 인터페이스 (Interface)**
    - 각 장비가 연결되는 단자를 의미하며, 포트마다 상태를 나타내는 **LED**가 존재
- **link-up / link-down**
    - 장비가 연결되면 link-up, 분리되면 link-down 상태
- **uplink**
    - 상위 스위치 또는 라우터와 연결된 포트

### 2. L2 Distribution 스위치

Distribution 스위치는 **여러 Access 스위치를 묶고, 상위 계층(라우터)과 연결하는 중간 허브 역할**을 한다.

#### 주요 특징

- **Access 스위치를 위한 스위치**
    - 여러 Access 스위치를 연결해 하나의 통합 네트워크 구조를 구성
- **상대적으로 고성능**
    - 더 많은 처리량과 포트 수를 지원

## L2 통신 방식: Broadcast vs Unicast

### 1. Broadcast

Broadcast는 **동일한 LAN 안의 모든 장비에게 데이터를 전송**하는 방식이다.

#### 특징

- Frame의 **수신 MAC 주소를 FF-FF-FF-FF-FF-FF** (48비트 전부 1)로 설정
- 하나의 송신자가 모든 수신자에게 데이터를 전송
- 모든 장치가 데이터를 수신한다.

#### 단점

- 브로드캐스트 트래픽이 많아지면 **네트워크 전체 성능 저하**
- 다른 장비들의 통신을 방해할 수 있다.
- 반드시 필요한 상황 외에는 사용을 최소화하는 것이 바람직하다.

#### 제어 방법

- **L2 Distribution 스위치**에서 브로드캐스트가 다른 구역으로 퍼지지 않도록 제한 가능

---

### 2. Unicast

Unicast는 **하나의 송신자가 하나의 수신자에게만** 데이터를 보내는 통신 방식이다.

#### 특징

- 목적지 MAC 주소를 **특정 장비의 MAC 주소로 지정**
- 다른 장비는 해당 통신을 무시한다.

#### 장점

- 불필요한 네트워크 부하 없음
- 다른 장비의 통신을 방해하지 않는다.
- 가장 일반적이고 효율적인 통신 방식이다.
